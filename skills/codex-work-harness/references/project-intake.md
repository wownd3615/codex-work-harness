# Project Intake

Before starting a new project, ask only the information needed to avoid rework.

Core questions:
1. What should be built?
2. Who will use it?
3. What painful problem does it solve?
4. What are the top 3 MVP features?
5. What is excluded from this version?
6. What files, data, or systems are required?
7. What counts as success?

Classify features as:
- Must: required for this version.
- Should: important but not required for first validation.
- Could: useful later.
- Won't: explicitly out of scope.

Do not repeat information the user already gave. For simple projects, reduce the interview to 1-3 questions. Confirm risky assumptions before coding.
