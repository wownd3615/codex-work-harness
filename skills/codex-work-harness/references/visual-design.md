# Visual Design

Use this reference for user-facing screens, dashboards, landing pages, visual polish, responsive UI, and design reviews.

## Before Building
- Identify the user and screen goal.
- Check existing `DESIGN.md`, design tokens, and components.
- Extract rules from references; do not copy a site or brand.

## Rules
- Operational tools should be quiet, organized, and scan-friendly.
- Landing pages should show the product, brand, person, or place in the first viewport.
- Apps should open directly into the usable experience.
- Use consistent colors, type scale, spacing, radius, shadows, and states.
- Use existing icon/component libraries.
- Verify desktop and mobile layouts when feasible.

## Avoid
- Default purple/blue gradients.
- Decorative card piles.
- Unnecessary background effects.
- Different button styles for the same action.
- Text or controls overlapping on mobile.
- Adding visual polish that makes the workflow slower.
