# Project Level

Simple: simple static site, MVP, local tool, or small app. Prefer easy explanation, few files, minimal code.

Connected: login, database, API, admin page, external service, or deployment. Prefer clear data flow, API/UI separation, tests, and practical error handling.

Scale: infrastructure, multiple services, permissions, CI/CD, Kubernetes, Terraform, or high-risk operations. Prefer design docs, security, operation, and impact analysis.

Level changes workflow depth, not the user's minimal-code rule.
