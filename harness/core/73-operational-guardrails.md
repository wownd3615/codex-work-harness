# Operational Guardrails

하네스는 문서만으로 완성되지 않는다. 반복 실수를 줄이려면 규칙을 실행 장치로 옮긴다.

## 역할 분리
- `AGENTS.md`: 항상 적용할 짧은 원칙
- `harness/00-start-here.md`: 필요한 문서만 고르는 라우터
- `skills/`: 필요할 때만 읽는 작업 절차
- `scripts/`: 사람이 까먹기 쉬운 점검 자동화
- Codex rules/hooks 예시: 위험 명령과 종료 점검 보조

## 적용 원칙
- 기본 저장소에는 비활성 예시만 둔다.
- 실제 프로젝트에 `.codex/` 설정을 켜기 전에는 사용자나 팀의 승인을 받는다.
- 위험 명령은 문장으로만 금지하지 말고, 가능한 경우 rules나 스크립트로 확인한다.
- hooks는 보조 장치다. hooks가 통과해도 최종 책임은 메인 작업자가 진다.

## 우선 자동화할 것
1. 하네스 파일 누락 점검
2. AGENTS.md 크기 점검
3. 위험한 삭제/이동/강제 push 전 확인
4. 완료 전 테스트/검증 보고 누락 확인
5. 오래된 명칭이나 깨진 경로 검색

## 주의
자동화가 너무 많으면 토큰보다 시간이 더 낭비된다. 작은 프로젝트는 문서와 스크립트만으로 충분하다.
