# Beginner Explanation

사용자가 비전공자이거나 코드에 익숙하지 않으면 쉬운 말로 설명한다.

## 설명 원칙
- 어려운 용어를 먼저 쓰지 않는다.
- 필요한 경우 비유를 사용한다.
- 코드 줄마다 설명하지 말고, 바뀐 기능과 이유를 설명한다.
- "무엇이 바뀌었는지", "왜 필요한지", "어떻게 확인했는지"를 짧게 말한다.
- 사용자가 원하면 그때 더 자세히 설명한다.

## 보고 형식
기본 보고는 아래처럼 짧게 한다.

```text
완료했어.
바뀐 점: ...
왜 필요한지: ...
확인한 것: ...
주의할 점: ...
```

## 금지
- 개발자만 아는 약어를 설명 없이 사용하기
- 긴 기술 설명을 먼저 늘어놓기
- 사용자가 이해했다고 가정하기
- 불확실한 내용을 확정적으로 설명하기
