# Start Here

Codex Work Harness는 Codex가 프로젝트를 매번 같은 기준으로 시작하고 끝내게 하는 작업 규칙이다.

## 핵심 철학
- 짧게 만든다.
- 필요 없는 코드는 지운다.
- 파일은 고치기 쉽게 나눈다.
- 필요한 것만 읽고 필요한 skill만 쓴다.
- 아이디어 단계부터 설계, 구현, 테스트, 보고까지 돕는다.

## 기본 로딩
항상 읽는 문서는 최소로 유지한다.

- `AGENTS.md`
- 이 파일

작업 전 반드시 기억할 원칙:
- 거짓말 금지
- 최소 코드
- 불필요 코드 삭제
- 필요한 파일/skill만 읽기
- 검증하지 않은 것은 검증했다고 말하지 않기

## 토큰 절약
상세 문서는 필요할 때만 연다. 모든 문서를 한 번에 읽지 않는다. 작은 작업은 `AGENTS.md`와 필요한 작업 문서 1-2개만으로 처리한다.
