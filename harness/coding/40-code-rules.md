# Code Rules

## 최우선 코드 원칙
요청한 기능을 만족하는 가장 짧고 단순한 코드를 작성한다.

## 작성 규칙
- 긴 코드보다 짧고 명확한 코드를 우선한다.
- 과한 추상화, 과한 구조화, 미래 대비용 코드는 만들지 않는다.
- 지금 요청한 기능에 직접 필요한 코드만 작성한다.
- 기존 코드로 해결 가능하면 새 코드를 만들지 않는다.
- 새 파일, 새 함수, 새 컴포넌트는 꼭 필요할 때만 만든다.
- 새 라이브러리는 꼭 필요할 때만 추가한다.
- 주석은 `coding/42-comment-rules.md` 기준에 따라 꼭 필요한 곳에만 단다.

## 삭제 규칙
- 수정/추가로 더 이상 필요 없어진 기존 코드는 삭제한다.
- 사용하지 않는 import, 변수, 함수, 컴포넌트, 파일을 정리한다.
- 주석 처리된 옛 코드와 임시 코드를 남기지 않는다.
- 삭제 전 `rg`로 참조 여부를 확인한다.
- 확실하지 않은 삭제는 삭제 후보로 보고한다.

## 파일 분리 규칙
- 코드는 짧게 쓰되 역할별로 분리한다.
- UI, 상태, API 호출, 데이터 처리, 설정값은 필요할 때 분리한다.
- 두 번 이상 쓰이거나 바로 재사용할 코드만 공용 함수나 컴포넌트로 분리한다.
- 한 번만 쓰이는 짧은 코드는 과하게 분리하지 않는다.
- 파일 이름만 봐도 역할을 알 수 있게 한다.
