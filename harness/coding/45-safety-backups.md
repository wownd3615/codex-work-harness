# Safety Backups

파일이 날아가거나 잘못 수정될 때를 대비해 되돌릴 수 있는 상태를 만든다.

## 기본 원칙
- 대규모 수정, 삭제, 이동 전에는 백업 또는 git 상태 확인을 한다.
- 사용자가 만든 변경은 절대 임의로 되돌리지 않는다.
- 삭제 전에는 참조 여부를 확인한다.
- 위험한 작업은 한 번에 크게 하지 않고 작은 단위로 나눈다.

## Git이 있는 프로젝트
작업 전 확인:

```powershell
git status --short
```

권장:
- 작업 전 커밋 또는 임시 브랜치 생성
- 큰 변경 전 `git diff` 확인
- 작업 후 변경 파일 목록 보고

금지:
- 사용자 승인 없는 `git reset --hard`
- 사용자 승인 없는 강제 push
- 사용자 변경 섞인 파일을 임의로 되돌리기

## Git이 없는 프로젝트
큰 수정 전 스냅샷을 만든다.

```powershell
powershell -ExecutionPolicy Bypass -File scripts/snapshot.ps1 -Root .
```

스냅샷은 `.codex-snapshots/` 아래에 저장한다. `node_modules`, `.git`, 빌드 결과물은 제외한다.

## 백업이 필요한 경우
- 파일 삭제
- 폴더 이동/이름 변경
- 대량 포맷팅
- 자동 생성 코드 덮어쓰기
- 의존성/설정 파일 변경
- DB 마이그레이션 또는 데이터 변경 스크립트

## 보고
백업을 만들었으면 경로를 보고한다. 백업하지 않았다면 이유를 말한다.
