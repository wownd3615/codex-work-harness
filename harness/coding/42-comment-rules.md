# Comment Rules

주석은 코드를 길게 만들지 않기 위해 신중하게 사용한다.

## 주석을 다는 경우
- 코드만 봐서는 의도를 알기 어려운 업무 규칙
- 보안상 중요한 처리
- CAD LSP에서 system variable 원복, undo, error 처리처럼 실수하면 위험한 부분
- 임시 우회 코드가 필요한 경우와 제거 조건

## 주석을 달지 않는 경우
- 코드가 이미 명확한 경우
- 함수 이름으로 충분히 설명되는 경우
- "값을 설정한다" 같은 당연한 설명
- 오래된 코드를 주석 처리해서 보관하는 경우

## 금지
- 안 쓰는 코드를 주석 처리해서 남기기
- TODO만 남기고 이유/조건을 쓰지 않기
- 주석과 실제 코드가 다르게 남기기

## 원칙
주석은 적게, 하지만 위험한 의도는 분명하게 남긴다.
