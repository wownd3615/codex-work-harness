# Done Checklist

완료 전 필요한 만큼 확인한다.

## 공통
- 하지 않은 일을 했다고 말하지 않았는가
- 보지 않은 파일/실행하지 않은 테스트를 확인했다고 말하지 않았는가
- 요청한 기능을 만족했는가
- 코드가 최소한으로 짧은가
- 과한 추상화가 없는가
- 새 파일/새 함수/새 라이브러리가 정말 필요했는가
- 더 이상 필요 없는 코드가 삭제되었는가
- 사용하지 않는 import/변수/함수가 없는가
- 파일 분리가 유지보수에 적절한가
- 기존 스타일을 따랐는가
- 확인한 사실, 추론, 불확실한 부분을 구분했는가
- 테스트/검증하지 않은 내용을 검증했다고 말하지 않았는가
- 삭제/대규모 수정 전 git 상태 또는 백업을 확인했는가

## 검증
- 가능한 테스트를 실행했는가
- 앱이면 로컬 실행 또는 브라우저 확인을 했는가
- 문서/슬라이드/스프레드시트면 렌더링 또는 열람 검증을 했는가
- 검증하지 못했다면 이유를 보고했는가

## 보안/사업성/UI/UX
- 로그인, 결제, 개인정보, 권한, 파일 업로드, 외부 API 키가 있으면 보안 점검을 했는가
- 새 서비스/기능이면 근거 기반으로 MVP와 사업 검증 필요성을 확인했는가
- 사용자 화면이 있으면 빈 상태, 로딩, 에러, 모바일 사용성을 확인했는가

## CAD LSP
- `.lsp`/AutoLISP 작업이면 명령어 이름과 도면 변경 동작을 보고했는가
- `.lsp` 파일 인코딩을 확인하고 기존 인코딩을 유지했는가
- 한글 프롬프트/주석이 있으면 UTF-8 저장으로 깨질 위험을 확인했는가
- 선택집합 nil 처리와 대상 entity type 확인이 있는가
- system variable을 변경했다면 원복하는가
- 도면 변경 명령에 undo/error 안전장치가 있는가
- 실제 CAD 실행 여부를 사실대로 보고했는가

## 외주 모드
- 요구사항 충족 여부를 확인했는가
- 검수 기준을 만족하는가
- 납품/인수인계에 필요한 설명이 있는가

## 팀 모드
- 팀 규칙/브랜치/PR 흐름을 지켰는가
- 변경 범위가 요청과 일치하는가
- 리뷰어가 볼 수 있게 검증 결과와 불확실성을 남겼는가
- secret, token, 개인정보를 남기지 않았는가
