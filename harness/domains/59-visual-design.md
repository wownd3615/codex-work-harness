# Visual Design

좋은 UI는 예쁜 장식보다 사용자가 바로 이해하고 반복해서 쓰기 편한 화면이다.

## 작업 전
- 사용자가 누구인지 확인한다.
- 화면의 목적을 한 문장으로 정한다.
- 기존 `DESIGN.md`, 디자인 토큰, 컴포넌트를 먼저 찾는다.
- 참고 디자인이 있으면 색상/간격/컴포넌트 규칙만 추출하고 그대로 베끼지 않는다.

## 디자인 방향
업무툴, 관리자, CRM, 파일 관리 프로그램:
- 조용하고 촘촘하게
- 정보 비교와 반복 작업 우선
- 큰 히어로, 장식 카드, 과한 배경 금지

랜딩/브랜드/포트폴리오:
- 첫 화면에서 대상이 바로 보여야 함
- 이미지나 실제 제품/장소/상태가 중심
- 가치 제안은 짧고 구체적으로

앱/도구:
- 첫 화면에서 바로 사용할 수 있어야 함
- 빈 상태, 로딩, 에러 상태를 준비
- 버튼과 입력 흐름을 줄임

## 고급스럽게 보이는 기준
- 색상 수를 줄이고 의미를 정한다.
- 글자 크기 위계를 명확히 한다.
- 여백은 일관된 단위로 쓴다.
- 버튼, 카드, 입력창의 radius와 높이를 통일한다.
- 아이콘은 기존 라이브러리를 쓴다.
- 모션은 짧고 목적 있게 쓴다.
- 실제 화면 캡처나 브라우저 확인으로 겹침을 본다.

## 금지
- 보라색/파란색 그라데이션을 기본값처럼 쓰기
- 둥근 카드만 계속 쌓기
- 장식용 배경, 장식용 아이콘 남발
- 페이지마다 다른 버튼 스타일
- 모바일에서 글자와 버튼 겹침 방치
- 폰트 크기를 화면 너비에 따라 무리하게 키우기
- 설명 문구로 기능을 때우기

## 완료 전 확인
- 데스크톱과 모바일에서 깨지지 않는가
- 핵심 행동 버튼이 명확한가
- 색상 대비와 focus 상태가 보이는가
- 같은 역할의 UI가 같은 모양인가
- 기존 디자인 기준에서 벗어난 부분이 있다면 이유가 있는가
