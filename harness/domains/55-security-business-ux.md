# Security, Business, UX Gates

AI 코딩은 보안, 사업성, UI/UX를 놓치기 쉽다. 아래 조건에서는 반드시 점검한다.

## Security Gate
다음이 있으면 Security Specialist를 사용하거나 같은 기준으로 직접 검토한다.

- 로그인/회원가입
- 결제/구독
- 개인정보
- 관리자 권한
- 파일 업로드
- 외부 API 키
- 배포/환경변수

체크:
- client 코드에 secret/API key가 노출되지 않는가
- 서버에서 입력 검증을 하는가
- 권한 체크가 서버에서 수행되는가
- 개인정보가 불필요하게 저장되지 않는가
- 에러 메시지가 내부 정보를 노출하지 않는가
- 업로드 파일 타입/크기 제한이 있는가

## Business Gate
새 서비스나 기능을 만들 때 확인한다.

- 누가 돈을 내거나 시간을 쓸 만큼 불편한 문제인가
- MVP가 그 문제를 직접 해결하는가
- 유료화/운영 비용/고객 확보 경로가 말이 되는가
- 경쟁 서비스와 차이가 한 문장으로 설명되는가
- 지금 만들 기능이 사업 검증에 꼭 필요한가

## UX Gate
사용자 화면이 있으면 확인한다.

- 처음 보는 사람이 다음 행동을 알 수 있는가
- 핵심 작업까지 클릭/입력이 너무 많지 않은가
- 빈 상태, 로딩, 에러 상태가 있는가
- 모바일에서 텍스트와 버튼이 깨지지 않는가
- 관리자/반복 작업 화면은 빠르게 훑어볼 수 있는가

## 우선순위
보안 위험은 기능 완성보다 우선한다. 사업성/UX 검토는 기능을 늘리기 위한 것이 아니라 MVP를 줄이고 명확하게 만들기 위한 것이다.
