# Development Pipeline

새 프로젝트를 처음부터 만들 때만 참고한다.

## 9단계
1. Schema: 용어와 데이터 구조
2. Convention: 코드 규칙
3. Mockup: 화면 흐름 검증
4. API: 백엔드/API 설계와 구현
5. Design System: 공용 UI 기준
6. UI Integration: 실제 화면과 API 연결
7. SEO/Security: 검색, 보안, 권한
8. Review: 구조, 코드, 품질 점검
9. Deployment: 배포와 운영

## 레벨별 적용
- Simple: 1, 2, 3, 6, 7, 9 위주
- Connected: 1부터 9까지 필요한 만큼
- Scale: 1부터 9까지 적용하되, 실제 위험이 있는 단계만 깊게 다룬다.

## 중요한 원칙
전체 프로젝트에 Planning Cycle를 한 번 적용하는 것이 아니다. 각 단계 안에서 작게 Plan, Design, Build, Review, Improve를 돌린다.

## 토큰 절약
이 문서는 새 프로젝트 시작, 개발 순서 고민, 큰 구조 설계 때만 사용한다.
