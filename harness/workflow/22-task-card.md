# Task Card

Task Card는 큰 문서까지는 필요 없지만, 바로 코딩하기에는 놓칠 것이 있는 작업에 쓴다.

목표는 생각을 길게 쓰는 것이 아니라, 작업 기준을 잃지 않는 것이다.

## 사용 기준
아래 중 하나라도 해당하면 Task Card를 만든다.

- 파일을 3개 이상 수정한다.
- 작업이 여러 단계로 나뉜다.
- 중간에 멈췄다가 이어받을 가능성이 있다.
- 서브에이전트가 필요하다.
- 삭제, 이동, 데이터 변경, 배포처럼 위험이 있다.
- 외주 검수나 팀 공유가 필요하다.

작은 수정, 오타 수정, 단일 함수 수정에는 쓰지 않는다.

## 위치
기본 위치:

```text
docs/00-task-cards/{task-name}.md
```

개인 실험이나 임시 작업이면 `.codex-notes/` 아래에 둘 수 있다.

## 최소 항목
`templates/task-card-lite.md`를 복사해서 사용한다.

- Goal: 무엇을 끝내야 하는가
- Context: 읽은 파일과 확인한 사실
- Constraints: 지켜야 할 조건
- Steps: 할 일
- Progress: 현재 상태
- Decisions: 결정과 이유
- Verification: 검증 방법과 결과
- Cleanup: 삭제하거나 정리한 코드

## 규칙
- 한 작업에는 Task Card 하나만 둔다.
- 사실과 추측을 섞지 않는다.
- 진행하면서 바뀐 결정은 지우지 말고 짧게 남긴다.
- 완료 후 보존 가치가 없으면 삭제하거나 보고서로 요약한다.
- Task Card 때문에 코드 작업이 느려지면 너무 무거운 것이다.
