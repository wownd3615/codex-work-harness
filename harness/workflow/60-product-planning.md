# Product Planning

아이디어 단계부터 Codex가 도와야 할 항목이다.

## 아이디어 정리
- 무엇을 만들지 한 문장으로 정리한다.
- 누가 쓰는지 정한다.
- 사용자가 얻는 이득을 정한다.
- 핵심 기능 3개를 고른다.
- 만들지 않을 것을 정한다.

## MVP 기준
MVP는 "가장 작지만 실제로 쓸 수 있는 버전"이다.

포함:
- 핵심 문제를 해결하는 기능
- 직접 눌러보고 확인 가능한 화면
- 저장/불러오기 등 필수 흐름

제외:
- 멋있지만 당장 필요 없는 기능
- 복잡한 관리자 기능
- 미래 확장용 구조
- 과한 애니메이션과 장식

## 외주용 산출물
- 요구사항 정의서
- 화면 목록
- 기능 목록
- 검수 기준
- 납품 파일 기준
- 유지보수/인수인계 기준
