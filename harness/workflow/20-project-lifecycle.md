# Project Lifecycle

현재 작업 단계를 먼저 분류한다.

## Idea
아이디어가 흐릿한 단계.

해야 할 일:
- 사용자가 만들고 싶은 결과를 한 문장으로 정리한다.
- 핵심 사용자, 핵심 문제, 핵심 기능 3개를 잡는다.
- 만들지 않을 것을 정한다.
- MVP 범위를 줄인다.

## Planning
무엇을 만들지 정하는 단계.

해야 할 일:
- 기능 목록을 MVP, 다음 버전, 제외로 나눈다.
- 외주라면 요구사항/납품기준/검수기준을 만든다.
- 직접개발이라면 개발 순서와 위험 요소를 정한다.

## Design
구조와 화면을 설계하는 단계.

해야 할 일:
- 화면 흐름을 정한다.
- 데이터 구조와 API 흐름을 정한다.
- 파일 구조를 단순하게 잡는다.
- 과한 기술 선택을 피한다.

## Build
구현 단계.

해야 할 일:
- 관련 파일만 읽는다.
- 최소 코드로 구현한다.
- 대체되어 필요 없어진 코드를 삭제한다.
- 필요한 검증을 한다.

## Review
완성 전 점검 단계.

해야 할 일:
- 버그 가능성
- 과한 코드
- 쓰레기 코드
- 테스트 누락
- 외주 납품 기준 충족 여부
