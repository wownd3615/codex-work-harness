# Work Mode

프로젝트 시작 때 먼저 사용자의 상황을 판단한다.

## Direct Mode
사용자가 직접 만들고 운영하는 프로젝트.

Codex는 다음을 우선한다.
- 빠른 MVP
- 이해 쉬운 구조
- 최소 코드
- 수정하기 쉬운 파일 분리
- 비전공자도 이해할 수 있는 보고

## Outsource Mode
사용자가 외주를 맡기거나 외주 결과물을 검토하는 프로젝트.

Codex는 다음을 우선한다.
- 요구사항 문서화
- 화면/기능 범위 명확화
- 납품 기준 정의
- 견적/일정 리스크 확인
- 결과물 검수 체크리스트
- 유지보수 인수인계 기준

## Unknown
모드가 불명확하면 먼저 합리적으로 추정한다. 위험한 계약, 비용, 일정 판단이 있으면 사용자에게 짧게 확인한다.

## Team Mode
여러 사람이 같은 저장소에서 작업하는 프로젝트.

Codex는 다음을 우선한다.
- 팀 규칙과 저장소 규칙 확인
- 브랜치/PR/리뷰 흐름 존중
- 변경 범위 최소화
- 충돌 가능성 확인
- 보안/secret/개인정보 노출 방지
- 변경 이유와 검증 결과를 PR에 남길 수 있게 정리

팀 프로젝트에서는 개인 취향보다 팀 합의 규칙이 우선한다.
